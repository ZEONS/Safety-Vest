# 반응형 웹(Responsive Web) 적용 계획서

기존 1920x1080 Full HD (스크롤 없음)에 맞춰져 있던 고정형 레이아웃을 다양한 화면 크기(노트북, 태블릿, 모바일)에 대응할 수 있도록 반응형(Responsive) 코딩을 적용합니다.

## User Review Required

> [!IMPORTANT]
> 화면이 좁아졌을 때(모바일/태블릿) 100개의 카드를 처리하는 방식에 대해 결정이 필요합니다.
> 현재 화면 비율에 맞춰 글자와 카드 크기를 현미경처럼 끝까지 작게 줄여서라도 **"스크롤 없이 무조건 한 화면에 100개"**를 보여줄지, 
> 아니면 모바일/작은 화면에서는 가독성을 위해 **그리드를 2열, 4열 등으로 변경하고 세로 스크롤을 허용**할지 결정해 주시기 바랍니다.
> 
> *추천:* 보통 모바일 및 태블릿에서는 글자가 너무 작아지면 모니터링이 불가능하므로, **특정 해상도(예: 1200px) 이하에서는 세로 스크롤을 허용하고 사이드바를 하단으로 내리는 방식**을 추천합니다.

## Open Questions

> [!WARNING]
> 사이드바(Event Log)의 배치: 모바일 화면(폭 768px 이하)일 경우 Event Log를 화면 가장 아래로 내릴까요, 아니면 팝업(혹은 탭) 형태로 숨길까요? 기본적으로는 화면 아래로 배치하는 것을 제안합니다.

## Proposed Changes

### [MODIFY] `style.css` (CSS Media Queries 적용)
- **Viewport Meta Tag 보완**: 모바일 줌(Zoom) 비율 대응 확인.
- **Breakpoints (미디어 쿼리)**:
  - **`@media (max-width: 1400px)`**: 노트북 환경. 폰트 크기 및 카드 여백 축소. 10x10 그리드 유지 (또는 8x13 등 비율 유연화).
  - **`@media (max-width: 1024px)`**: 태블릿 환경. 사이드바(Event Log) 영역을 가로폭의 25%에서 하단 또는 우측 축소형으로 변경. 카드 그리드는 `grid-template-columns: repeat(auto-fit, minmax(120px, 1fr))`로 변경하여 스크롤 허용.
  - **`@media (max-width: 768px)`**: 모바일 환경. Header 통계를 2줄 이상으로 감싸기(Wrap). 사이드바를 하단으로 완전 이동. 카드 크기를 모바일에 최적화.
- **가변 단위 적용**: 고정된 픽셀(px) 대신 `rem`, `%`, `vw/vh` 등을 조합하여 브라우저 크기에 따라 레이아웃이 유동적으로 변하도록 CSS 재설계.
- **모달(Modal) 창 반응형**: 모바일 기기에서 팝업 차트가 화면을 꽉 채우도록 크기 조절.

## Verification Plan

### 브라우저 리사이징 테스트
- Chrome 브라우저의 개발자 도구(Device Toolbar)를 사용하여 1920px(기존), 1366px, 1024px(iPad), 390px(iPhone)에서 UI가 깨지지 않고 정상적으로 렌더링되는지 수동으로 검증합니다.
- 창 크기를 마우스로 줄이거나 늘릴 때 레이아웃 붕괴 현상이 발생하지 않는지 확인합니다.
